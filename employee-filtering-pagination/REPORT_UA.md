# Протокол виконання лабораторної роботи

## Тема

Технічні засоби реалізації безпечної авторизації у корпоративних розподілених системах засобами Spring.

## Індивідуальне завдання

Варіант 4 — додавання фільтрації та пагінації для списку працівників.

## Мета роботи

Реалізувати захищений REST endpoint для отримання списку працівників із пагінацією, сортуванням та фільтрацією за посадою і діапазоном зарплати, використовуючи Spring Data JPA та Spring Security.

## Реалізація

Сутність `Employee` містить ім'я, прізвище, посаду та базову зарплату. Репозиторій `EmployeeRepository` розширює `JpaRepository` і `JpaSpecificationExecutor`. Це дозволяє створювати динамічні умови пошуку за допомогою класу `EmployeeSpecifications`.

Endpoint:

```text
GET /api/employees
```

Підтримувані параметри: `position`, `minSalary`, `maxSalary`, `page`, `size`, `sortBy`, `direction`.

Для некоректних параметрів пагінації застосовується нормалізація: від'ємний номер сторінки замінюється на `0`, нульовий або від'ємний розмір — на `10`, а значення понад `100` обмежується до `100`.

Доступ до `/api/employees/**` дозволено лише автентифікованим користувачам. У навчальній реалізації використано HTTP Basic із користувачем `admin` та паролем `password`.

## Результати тестування

Було реалізовано п'ять інтеграційних тестів:

1. Пагінація повертає обмежену кількість записів.
2. Фільтрація за посадою повертає лише відповідні записи.
3. Комбінована фільтрація одночасно враховує посаду, мінімальну та максимальну зарплату.
4. Некоректні параметри пагінації замінюються безпечними значеннями.
5. Неавторизований запит отримує HTTP 401, а авторизований — HTTP 200.

## Висновок

У ході роботи було реалізовано захищений REST API зі Spring Data JPA Specifications, пагінацією та фільтрацією. Використання DTO відокремлює зовнішній контракт API від JPA-сутності, а Spring Security обмежує доступ до даних працівників. Модульні тести підтвердили коректність основних сценаріїв роботи системи.
